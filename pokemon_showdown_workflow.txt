HOW TO MODIFY WORKFLOW TO USE PLAY.POKEMONSHOWDOWN.COM

1.  REMOVE LOCAL SERVER SETUP Delete the entire “Start Pokemon Showdown
    server” step from your workflow YAML.

2.  UPDATE WEBSOCKET URL IN YOUR CODE Replace:
    ws://localhost:8000/showdown/websocket

With: wss://sim3.psim.us/showdown/websocket

3.  ADD ENV VARIABLE (OPTIONAL BUT RECOMMENDED) In your workflow:
    SHOWDOWN_URL=wss://sim3.psim.us/showdown/websocket

In Python: url = os.getenv(“SHOWDOWN_URL”,
“ws://localhost:8000/showdown/websocket”)

4.  IMPLEMENT LOGIN (REQUIRED FOR STABLE USE)

-   Request assertion:
    https://play.pokemonshowdown.com/action.php?act=getassertion&userid=USERNAME

-   Send via WebSocket: /trn USERNAME,0,ASSERTION

5.  LIMITATIONS OF PUBLIC SERVER

-   Rate limits
-   Possible bans
-   No custom formats
-   Slower training

6.  RECOMMENDATION Keep local server for training. Use public server
    only for evaluation/testing.
